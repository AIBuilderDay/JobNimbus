# backend/CLAUDE.md

Backend conventions. Read this before editing anything under `backend/`. Repo-level conventions live in [../CLAUDE.md](../CLAUDE.md).

Stack: Python 3.14, FastAPI, `uv`, `httpx`, Pydantic v2, `pydantic-settings`, `python-json-logger`, stdlib `sqlite3`. Tests via `pytest` + `pytest-asyncio` + `respx`.

## Env vars — only via `settings`

```python
from settings import settings
api_key = settings.GOOGLE_MAPS_API_KEY
```

- Never `os.environ.get(...)` outside [settings.py](settings.py). Add new vars to the `Settings` class **and** [.env.example](.env.example) in the same edit.
- `.env.example` uses `op://AIBuilderDay/<item>/credential` references. Local dev resolves them via `task backend:dev` (which runs the container with `--env-file .env`) or `op run --env-file=.env -- <cmd>` for one-off scripts. `.env` is gitignored and regenerated by `task env:generate`.
- Default values go on the `Settings` field, not in callsites. Tests instantiating `Settings` directly should pass `_env_file=None` and `monkeypatch.delenv` per-var to stay isolated.

## Logging — `get_logger(__name__)`, never `print`

```python
from logger import get_logger
log = get_logger(__name__)

log.info("geocoding address=%s", address)
```

- One logger per module, named `__name__`.
- Log on entry **and** exit of every external call. Include the inputs that matter (address, lat/lng, status code).
- Inside `except`, use `log.exception(...)` — it captures the traceback automatically. Do not pass the exception object as a `%s` arg.
- No `print()` in production code. Scripts under `scripts/` may print since their output is the user-facing report.

## Try / except — wrap every external call

```python
try:
    async with httpx.AsyncClient(timeout=10.0) as client:
        resp = await client.get(url)
    resp.raise_for_status()
except httpx.HTTPError:
    log.exception("eagleview fetch failed for address=%s", address)
    raise EagleViewError("upstream failed")
```

- Catch the **narrowest** exception that fits (`httpx.HTTPError`, `ValidationError`, `sqlite3.IntegrityError`). No bare `except:`.
- Re-raise as a typed app exception or return a sentinel (`None`, empty list). Never swallow silently.
- Known status codes that aren't errors (e.g. Solar's `404 = no coverage`) are checked **before** `raise_for_status()` and returned as a domain value, not an exception.

## Async + httpx

- All providers/services are `async def`. Routers can be sync or async — match the surrounding code.
- `httpx.AsyncClient(timeout=10.0)` for normal calls; bump to `timeout=30.0` for slow EagleView submissions.
- Always `raise_for_status()` unless you're explicitly handling a known non-2xx code.

## Pydantic models = the contract

Layers exchange Pydantic models, not dicts. Services accept and return models. Only the DAO converts to/from primitives for SQLite.

- Pydantic v2 syntax only: `model_config = ConfigDict(...)`, `@model_validator(mode="after")`, `Field(default_factory=...)`. No `class Config`, no `@validator`.
- Make domain models `frozen=True` when they represent immutable facts (e.g. an `Address`); use `model_copy(update=...)` to derive new instances.
- Money in **cents** (`int`), never floats. Validate `quantity * unit_price ≈ total` to within a 1¢ tolerance.
- Store the upstream raw payload on the model (`raw: dict = Field(default_factory=dict)`) so we never re-fetch to inspect a field we forgot to map.

## Layering — provider / service / dao / router

- **Provider** ([providers/](providers/)): wraps one external API. Stateless. No business logic. Returns models or dicts.
- **Service** ([services/](services/)): business logic. Composes providers + DAOs. Returns models.
- **DAO** ([dao/](dao/)): SQLite reads/writes. No business logic. One module per table.
- **Router** ([routers/](routers/)): parse request → call service → return response. Thin.

Do not call providers from routers. Do not put SQL in services.

### DAO specifics

- `sqlite3` from stdlib, with `row_factory = sqlite3.Row`.
- Open and close connections per-call (a context manager is fine for our volume).
- JSON blobs are fine for nested data — no migrations during the hackathon.
- All DDL lives in [dao/database.py](dao/database.py) `init_db()` using `CREATE TABLE IF NOT EXISTS`.

## Critical bug — roof area ≠ footprint

Submitted square footage must be **slanted roof area**, not projected footprint. Pitch multiplier:

```python
import math
def pitch_multiplier(rise: int, run: int) -> float:
    return math.sqrt(1 + (rise / run) ** 2)
# 4:12 → 1.054   6:12 → 1.118   8:12 → 1.202
```

Google Solar's `stats.areaMeters2` is **already slanted** — do not multiply again. Only EagleView's footprint values (or any other projected area) need the pitch multiplier. `Measurement.apply_pitch_multiplier(rise, run)` is idempotent on purpose.

## Tests

- Live in [tests/](tests/), mirroring source structure.
- `pytest` + `pytest-asyncio`. Run via `task backend:test`.
- Mock HTTP with `respx`; mock async functions with `unittest.mock.AsyncMock`.
- Every service method gets a unit test. Every provider gets a happy-path + 1 error test. Validation scripts (e.g. [scripts/validate_models_vs_api.py](scripts/validate_models_vs_api.py)) double as integration tests.
- Don't merge with red tests.

## Hackathon constraints

- SQLite is **ephemeral** on Render — DB resets every deploy. Don't store anything we can't reproduce. EagleView cache is rebuildable from the precache script.
- No DB migrations. Only `CREATE TABLE IF NOT EXISTS` in `init_db()`.
